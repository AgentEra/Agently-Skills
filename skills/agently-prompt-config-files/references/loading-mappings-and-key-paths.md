# Loading, Mappings, And Key Paths

This page covers how prompt config is loaded and how placeholders are resolved.

## 1. File Path Or Raw Content

Both loaders accept either:

- a real file path
- raw YAML / JSON text

Current behavior:

- if the argument points to an existing file, Agently reads the file
- otherwise Agently parses the argument as prompt content

That makes these use cases equally valid:

- local prompt files in a repository
- prompt text stored in a database
- prompt text fetched from a remote config service

## 2. YAML And JSON Loader Behavior

### YAML

- `load_yaml_prompt(...)` uses `yaml.safe_load`

### JSON

- `load_json_prompt(...)` uses `json5`
- that means comments and trailing commas are accepted on the current path

This is useful when prompt config is edited by humans instead of generated by a strict serializer.

## 3. `mappings`

`mappings` are applied while prompt keys and values are written into prompt state.

That means mappings can affect:

- prompt values
- placeholder-based key names
- both agent-side and request-side routed entries

Example outcome:

- `$${agent_key}` can become an agent prompt key such as `extra_info`
- `${request_key}` can become a request prompt key such as `extra_request`

Important boundary:

- `.alias` parameters do not receive this same load-time mapping substitution

## 4. `prompt_key_path`

Use `prompt_key_path` when one YAML or JSON document contains multiple prompt packs.

Example shape:

```yaml
packs:
  support:
    .request:
      instruct: Answer as support.
  tutor:
    .request:
      instruct: Answer as a tutor.
```

Then load one branch:

```python
agent.load_yaml_prompt(content, prompt_key_path="packs.support")
```

The selected path must resolve to a dictionary prompt config, not a scalar or list.

## 5. Load-Time Materialization

Mappings are resolved during load, not deferred.

So after:

- loading a template with mappings
- exporting it again through `get_yaml_prompt()` or `get_json_prompt()`

the exported prompt contains the materialized prompt state, not the original placeholder template.
